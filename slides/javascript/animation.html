Why use animation and transition

The big question is why use animation? There are many reasons why people use animation in thier sites, but one use case for adding animation is that subtle animation can help the end-user navigate your site and guide the end-user attention to certain areas.[Show Example] Visual appeal is another use-case. Animation done well can engage the end-user and even provide some interactivity. Lets take a look at an example. An over-use of animation can distract and even cause end-users to be confused.

CSS Transitions

Transition – is a change of state from an initial state to an end-state. CSS transitions allows you to change property values smoothly (from one value to another), over a given duration.

transition: all 3s; is the same as transition-property: all; transition-duration: 3s;

Animatable Properties

  • width, border-width, right, color, top, opacity, padding, background-color, font-size, line-height, opacity, left, margin, height, bottom
  • These properties typically have a numeric measurement value with can change in the intermediate state.

@Keyframes rule

The @keyframes CSS at-rule controls the intermediate steps in a CSS animation sequence by defining styles for keyframes (or waypoints) along the animation sequence

								
									@keyframes movingbox {
									    0%   {top: 0px; left: 0px; background: red;}
									    25%  {top: 0px; left: 500px; background: blue;}
									    50%  {top: 500px; left: 500px; background: yellow;}
									    75%  {top: 500px; left: 0px; background: green;}
									    100% {top: 0px; left: 0px; background: red;}
									}
									div {
									    width: 100px;
									    height: 100px;
									    background: red;
									    position: relative;
									    -webkit-animation: movingbox 5s infinite; 
									}
